Source snippets

03

When you click an element, Inspekt fetches the surrounding lines and renders them in the popover. Resolves via the dev server's /__inspekt/snippet endpoint when @aylith/inspekt-vite is running, or via deployed source maps if you opt in.

Display

Lines of context Lines above and below the target line (max 30).

Source-map fallback advanced

When enabled, Inspekt fetches .map files from your app's bundle URLs when the local dev server isn't running. Useful for staging/production sites or extension pages.

Privacy note. Fetching .map files makes additional network requests visible in your DevTools and in corporate proxy/firewall logs. Map files can be multi-MB. Disable this on shared or monitored networks.

Agents & toolbar icon

04

Inspekt registers an MCP server with the agents you have installed via npx inspekt setup. The toolbar icon describes per-tab capability at a glance.

Toolbar icon legend

  • · Greyscale — page has no Inspekt instrumentation; the inspector won't find any sources here.
  • OFF Page is instrumented but the inspector is disabled. Click the icon to enable.
  • ON Active — file path only (no snippet source available).
  • DEV Active — snippets via the local dev server.
  • MAP Active — snippets via source maps (opt-in fallback).
  • AI Active — agent connected via MCP. Grabs route to your editor's agent.

Setup

To register Inspekt with your installed agents, run: 

npx inspekt setup

Detects every agent on your machine and writes an MCP server entry to each. Idempotent.
